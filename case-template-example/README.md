# Шаблон нового кейса Kovtun.studio

Эта папка не подключена к сайту автоматически. Она предназначена для копирования и подготовки материалов нового проекта.

## Как пользоваться

1. Скопируйте папку `case-template-example` целиком.
2. Переименуйте копию по проекту: латиницей, маленькими буквами, слова через дефис. Пример: `alaska-batteries`.
3. Заполните `case.md`. Всё в квадратных скобках нужно заменить.
4. Сравните результат с `case-example-filled.md` — это полностью заполненный пример.
5. Замените все файлы в `images/`, сохранив понятные имена.
6. Заполните `images/alt-text.md`.
7. Пройдите `checklist.md`.
8. Отправьте Codex путь к готовой папке и текст из `handoff-prompt.md`.

## Структура папки

```text
название-проекта/
├── README.md
├── case.md
├── case-example-filled.md
├── checklist.md
├── handoff-prompt.md
└── images/
    ├── README.md
    ├── alt-text.md
    ├── cover.webp
    ├── 01-overview.webp
    ├── 02-system.webp
    ├── 03-detail.webp
    ├── 04-application.webp
    └── 05-result.webp
```

## Допустимые направления

- `packaging` — Упаковка.
- `identity` — Логотип и фирменный стиль.
- `websites` — Сайты.
- `presentations` — Презентации.
- `car-wrap` — Оклейка авто.

## Допустимые сегменты бизнеса

- `food` — FMCG и продукты питания.
- `automotive` — Автомобильный бизнес.
- `b2b` — B2B и корпоративный сектор.
- `creative-services` — Креативные и профессиональные услуги.

Если нужного сегмента нет, не выбирайте приблизительный. Напишите новый сегмент словами в `case.md` — при добавлении проекта он будет согласован и внесён в фильтр.

## Главное правило

Не придумывайте результаты, цифры и роли. Если информации нет, пишите `нет данных`. При командной работе перечисляйте реальный вклад всех участников.
